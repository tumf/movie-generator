## Context
LLMモデルが関数デフォルトで指定されており、設定ファイルの値が明示的に使われる保証がない。

## Goals / Non-Goals
- Goals:
  - すべてのLLM呼び出しでモデルを明示的に渡す
  - デフォルト値のハードコードを排除する
- Non-Goals:
  - モデル選択ロジックの変更や新規モデルの追加

## Decisions
- 関数シグネチャから `model` のデフォルトを削除し必須引数にする
- 呼び出し元は設定ファイルのモデル値を必ず渡す

## Risks / Trade-offs
- 呼び出し漏れがあると実行時エラーになるため、網羅的な更新が必要

## Migration Plan
- 既存の呼び出し経路を全て更新し、テストで漏れを検出する
