# Change: 台本生成品質の向上（ストーリーテリング・役割分担・内容理解の強化）

## Why

現状の台本生成には以下の品質課題があります：

1. **導入が唐突で視聴者を引き込めない** - いきなり本題に入り、背景説明や動機付けが不足
2. **話の筋が単調** - セクション間の論理的つながりが弱く、起承転結の構造が不明確
3. **複数話者の役割分担が中途半端** - 「解説役・質問役」程度の抽象的な指示のみで、自然な掛け合いを生む仕組みが不足
4. **内容の理解が浅い** - LLMが表面的な要約にとどまり、視聴者視点の疑問や具体例が欠如

これらは視聴維持率の低下や情報伝達効率の悪化につながります。

## What Changes

### 1. ストーリーテリング構造の強化

- **フック（掴み）セクションの必須化**: 冒頭で視聴者の興味を引く要素を明示
- **起承転結の構造化**: 導入→展開→転換→結論の論理的流れをプロンプトに組み込み
- **セクション間の接続詞指示**: 各セクションの役割（Why→What→How）を明示

### 2. 複数話者の役割設計の高度化

- **役割パターンの提供**: 具体的な会話パターンを例示（理解確認・疑問提起・具体例要求）
- **話者交代のリズム指示**: 単調な交互発言を避け、自然な会話流れを生成
- **個性の引き出し方**: キャラクター設定を活かした反応・リアクションの指示

### 3. 内容理解の深化

- **視聴者視点の疑問予測**: 「初めて聞く人が疑問に思うこと」を明示的に組み込む指示
- **具体例・比喩の必須化**: 抽象的な説明に具体例を加えるよう指示
- **段階的な説明指示**: 「概要→詳細→応用」の3段階構造

### 4. プロンプトの構造化

- **セクション構成テンプレート**:
  - Hook（0-15秒）: 問題提起・驚きの事実
  - Context（15-30秒）: 背景・なぜ重要か
  - Core Content（本編）: 3-5のメインポイント
  - Conclusion（まとめ）: 要点の再確認・次のアクション
- **チェックリスト追加**: LLMに「視聴者は理解できるか？」「興味を持てるか？」を自己評価させる指示

## Impact

### Affected specs
- `script-generation`: プロンプトテンプレート全面改訂（MODIFIED）

### Affected code
- `src/movie_generator/script/generator.py`: 4つのプロンプト定数を更新
  - `SCRIPT_GENERATION_PROMPT_JA`
  - `SCRIPT_GENERATION_PROMPT_EN`
  - `SCRIPT_GENERATION_PROMPT_DIALOGUE_JA`
  - `SCRIPT_GENERATION_PROMPT_DIALOGUE_EN`

### Breaking Changes
なし（プロンプト内容の改善のみ、APIや構造は変更なし）

## Success Criteria

1. **視聴者エンゲージメント**: 生成された台本を人間評価者がレビューし、「続きを見たい」と思える導入部が80%以上
2. **内容理解度**: 台本から生成した動画を初見の視聴者5名に見せ、主要ポイントの理解度が70%以上
3. **会話の自然さ**: 複数話者モードで生成した台本の会話が「棒読み感がない」と評価される割合が80%以上
